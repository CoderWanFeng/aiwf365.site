# 全自动播客流水线

> 选题 → 脚本 → TTS → 剪辑，每周一更零加班。

## 一、整体架构

```
RSS 源 ─→ 选题 Agent ─→ 脚本 Agent ─→ TTS ─→ 剪辑 ─→ RSS / 小宇宙
                       ↑
                  主持人人设 prompt
```

所有节点用 **n8n** 串起来，零代码。

## 二、Step 1：选题

抓取源头：

- 36Kr / 虎嗅 / 少数派的 RSS
- arXiv 当周热门 cs.AI 论文
- 小宇宙热门播客 TOP 50

扔给 **GPT-4o** 选 3 个最值得聊的，prompt 核心：

```text
你是播客主理人，从下面候选新闻里挑 3 个最有讨论价值的，
要求：和 AI / 创作者经济相关，且本周没人做过。
输出：标题 / 一句话钩子 / 3 个讨论方向。
```

## 三、Step 2：脚本

拿到选题后，让 **Claude Sonnet** 写脚本：

```text
基于选题 {title}，写一段 15 分钟双人播客脚本。
主持人：A（冷静理性），B（脑洞大）。
结构：开场闲聊 2 分钟 → 主题展开 8 分钟 → 对立观点 3 分钟 → 总结 2 分钟。
要求：每 30 秒至少一个笑点或金句。
```

输出格式固定为：

```markdown
## A: ...
## B: ...
```

## 四、Step 3：TTS

我用的是 **ChatTTS** + **GPT-SoVITS** 双引擎：

- A 用 ChatTTS 默认男声（理性）
- B 用 GPT-SoVITS 训练的女声（脑洞）

声音克隆样本只需要 5 分钟干净人声即可。

## 五、Step 4：剪辑

剪映「文生音频」自动卡点。背景音乐用 Epidemic Sound 的 Lo-fi 分类，-18dB 压底。

## 六、发布

导出 mp3 + 封面 → 自动推送到：

- 小宇宙（手动，因为要审核）
- Apple Podcasts（GitHub Action 定时触发）
- 公众号（图文版由脚本同步生成）
